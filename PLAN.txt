RocketPHP — Roadmap
===================

A Filament-style admin panel framework for Inertia.js + React.
Package: maherelgamil/rocketphp
Namespace: MaherElGamil\Rocket

Current status (CRUD + panel UX)
-------------------------------
- Panel + PanelManager (fluent config, auto-registers routes on register)
- Resource abstract: ListRecords, CreateRecord, EditRecord; store/update/destroy
- Form + field components: TextInput, Textarea, Select, Toggle, DatePicker,
  FileUpload; validation serialized to Inertia
- Table + Column base + TextColumn + BadgeColumn; search, sort, pagination;
  table filters (Select, Ternary, DateRange, Trashed); row + bulk actions
- ResourceController + optional row/bulk action routes; dashboard route + widgets
- Laravel Gate/Policy integration (viewAny, create, update, delete); nav filtered
- Self-contained React: rocket.tsx, list/create/edit, dashboard, data-table,
  panel-shell, alert dialog, form fields
- Blade root view: rocket::app; config/rocket.php
- Pest feature tests (resources, forms, auth, actions, filters, dashboard)


Next steps (higher-level features)
----------------------------------
1. Global search across resources (e.g. Ctrl+K palette), policy-filtered
2. Relation managers (Filament-style nested tables on edit pages)
3. Notifications / toasts: ship minimal flash UI or document peer dependency
4. Theming: CSS vars vs full design-token system
5. i18n: publish-friendly lang files per panel
6. Multi-tenancy helpers (optional stancl/tenancy integration)
7. Publish + real-app smoke test; tag releases

Recommended order
-----------------
Ship complex UX (relations, global search) after stabilizing host-app
integration. Keep README/CHANGELOG in sync when public APIs change.


Open design questions
---------------------
See "Next steps" above; same themes as before, now that core CRUD and panel
landing page exist.
