# Internationalization & RTL

RocketPHP ships with a JSON-based translation layer, a header locale
switcher, and automatic RTL support for scripts like Arabic and Hebrew.

## Enable the locale switcher

Configure the panel with a default locale and the list of locales to show
in the switcher:

```php
return $panel
    ->locale('en')
    ->availableLocales(['en', 'ar']);
```

The switcher appears in the panel header and persists the user's choice
via the `rocket.locale` route.

## Translations

Rocket bundles translations as JSON files under `lang/{locale}.json`.
Publish them to override any string in your host app:

```bash
php artisan vendor:publish --tag=rocket-lang
```

This copies the package's lang files into `lang/vendor/rocket/{locale}.json`.
Strings published here take precedence over the bundled defaults.

## Translating resource strings

On the PHP side, wrap user-facing labels with Laravel's `__()` helper so
the published lang file can override them:

```php
TextColumn::make('name')->label(__('Name'));
```

On the React side, use the `__()` helper exported from `@rocket/lib/i18n`:

```tsx
import { __ } from '@rocket/lib/i18n';

<Button>{__('Save')}</Button>
```

## RTL

When the active locale is RTL (e.g. `ar`), the panel shell sets `dir="rtl"`
at the root. The layout uses Tailwind **logical properties**
(`ms-*`, `me-*`, `ps-*`, `pe-*`) rather than physical ones (`ml-*`, `mr-*`),
so spacing, borders, and icon placement flip automatically.

When adding custom components:

- Use `ms-` / `me-` / `ps-` / `pe-` instead of `ml-` / `mr-` / `pl-` / `pr-`
- Use `start-*` / `end-*` instead of `left-*` / `right-*`
- Use `text-start` / `text-end` instead of `text-left` / `text-right`

This is the only convention you need to follow for full RTL support.
